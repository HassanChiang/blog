## 1.2 程序被其他程序翻译成不同的格式

``` c
#include <stdio.h>

    printf("Hello, world\n");
    return 0;
}
```

gcc 编译4阶段:

```
hello.c(文本) -> 预处理器(cpp) ->
hello.i(文本) -> 编译器(ccl) -> 
hello.s(文本) -> 汇编器(as) -> 
hello.o(二进制) + prntf.o(二进制) -> 连接器(ld) -> 
可执行文件：hello(二进制)
```

### 1.7.1 进程

操作系统抽象：`进程`、`虚拟内存`和`文件`。
进程，交错执行，称为`并发运行`。交错执行的机制称为`上下文切换`。
进程`上下文`：操作系统保持跟踪进程运行所需的所有状态信息，包括：PC(程序计数器)、寄存器值和主存内容等。

### 1.7.3 虚拟内存

`虚拟内存`

每个进程看到的内存都是一致的，称为`虚拟地址空间`。
虚拟内存空间由大量准确定义的`区`构成，包括：
* 程序代码和数据
* 堆
* 共享库
* 栈
* 内核虚拟内存
